# Pre-Meeting Brief — Phase 2 Implementation Plan (Agentic Pipeline + Admin Trigger)

> **For agentic workers:** Use superpowers:subagent-driven-development to execute task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Ship the agentic pipeline end-to-end — 4 real agents (Qualification, Research, Data Quality, Synthesis), deterministic merge with priority chains + 6 conflict detectors, real Claude calls via the LiteLLM proxy, and a password-gated admin page where the reviewer can type a domain and watch a brief generate live in 30-60 seconds.

**Architecture:** LangGraph orchestrator with `BriefState` (pydantic) flowing between nodes. Trigger → Qualification → resolve → fetch_all (4 sources parallel) → merge → synthesize → render. Each agent is a bounded async function with a Claude Sonnet 4.6 backbone. Web research uses Claude's native `web_search_20250305` tool. specter-mcp standalone server is wired as a real MCP tool call (subprocess + stdio transport).

**LLM stack:** Anthropic SDK → LiteLLM proxy at `api.mercury.weather.com/litellm` (verified working with `claude-sonnet-4-6` + `web_search` tool). Both `ANTHROPIC_API_KEY` and `ANTHROPIC_BASE_URL` env vars are already bound on Vercel.

**Tech additions:** `langgraph>=0.2.0`, `anthropic>=0.30.0`.

**Scope:** Phase 2 covers all 12 tasks below. Phase 3 (Vercel Cron + PDF + GitHub auto-deploy + eval rubric) is a separate later plan.

## Risks & known concerns

1. **Proxy quota**: LiteLLM proxy at api.mercury.weather.com is the user's corporate proxy. Cost goes to their company tab. Bound usage: each brief ≈ 4 Claude calls × ~$0.10 each ≈ $0.40. Set hard `max_iterations` and `wallclock` bounds.
2. **Web search tool quirks**: verified working through proxy, but result quality varies. Research Agent must gracefully handle empty results.
3. **MCP subprocess on Vercel**: Vercel Python functions run in a constrained sandbox. Launching subprocess works in theory but stdio MCP transport may not. **Fallback**: if subprocess launch fails, use in-process SpecterProvider — orchestrator code identical.
4. **Cold start**: full pipeline + Anthropic API + Postgres query ≈ 30-60s. Within Vercel Fluid Compute's 800s ceiling but feels slow. Stream status updates via polling.
5. **Stability**: 4 agents x N edge cases = many failure modes. Each agent must catch its own exceptions and write to `etl_run_log.error_message` rather than raise. The orchestrator surfaces partial brief output if some agents failed.
